# Sitemap 分析報告與建議建議

針對目前的網站地圖（Sitemap），我從 **UX 使用者體驗**、**架構完整性** 以及 **資料庫一致性** 三個維度提供了這份分析報告。

## 🔍 優點分析

1. **結構清晰**：主選單分類明確，使用者可以快速在「尋找旅程」、「查看心得」與「個人管理」之間切換。
2. **角色職責分明**：在會員中心區分了「主辦」與「參加」兩個邏輯，這對於具有「媒合」性質的平台非常重要。
3. **心得牆獨立**：將心得牆放在第一層導覽，有助於建立平台的可信度與社群氛圍。

## ⚠️ 關鍵建議 (缺口檢查)

### 1. 補足「管理員後台」頁面

雖然需求中有提到管理員（Admin）的角色，但目前 Sitemap 未顯示管理路徑。

- **建議新增**：`6. 管理員後台`（僅管理員可見）。
  - `6-1. 使用者管理`（封鎖/搜尋）。
  - `6-2. 旅程審核/管理`（下架違規內容）。
  - `6-3. 檢舉案件處理`。

### 2. 「加入旅團」的流程細化

目前 `3-1-1-1. 加入旅團` 之後的動作較不明確。

- **建議建議**：整合「自我介紹填寫」頁面。因為資料庫中有 `applications.intro` 欄位，這應該是一個獨立的彈窗或頁面流程。

### 3. 「聊聊/通知」的整合

目前的地圖將「聊聊/通知」放在會員中心深處。

- **建議建議**：通知（Notifications）通常需要一個全域的狀態（如導覽列的小鈴鐺），而不僅僅是放在會員中心的一個子頁面。

### 4. 登入後首頁的變化

- **建議建議**：首頁可以根據登入狀態變動。訪客看到「探索」，會員看到「推薦旅程」或「即將出發的叮嚀」。

## 🔄 與資料庫 (Schema) 的同步檢查

- **行程項目 (Itineraries)**：目前 Sitemap 似乎未明確標出「編輯/查看具體行程」的頁面，建議在「旅程詳情」或「發起旅程」的子步驟中強化此塊。
- **軟刪除顯示**：在會員中心的「分享紀錄」或「我發起的旅程」中，需要考慮是否顯示「已取消/已刪除」的歷史資料（對應用戶提到的軟刪除）。

## 💡 下一步行動建議

我們可以優先討論 **「首頁」** 與 **「旅程詳情頁」** 的元件組成。這兩個頁面決定了平台的第一印象與核心功能。
