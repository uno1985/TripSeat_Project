# tripSeat UI/UX 設計指南

本指南根據設計師提供的首頁樣板，分析其視覺語彙並轉化為開發規格。

## 🎨 視覺風格定位 (Visual Language)

- **色調**：以 **明亮橘/黃 (#FBB03B)** 為主色，搭配大量留白 (White Space)。
- **插畫風格**：採用簡約的黑白線條插畫，賦予品牌一種輕鬆、友善且具有溫度的親和感。
- **字體與氛圍**：現代黑體字型，強調「跟著旅程走就對了」的隨性與信任感。

## 🧩 核心元件拆解 (Component Breakdown)

| 元件名稱 | 功能描述 | 技術重點 |
| :--- | :--- | :--- |
| **Hero Section** | 強調標語與插畫，引導用戶開始探索。 | 需支援 RWD 縮放，確保插畫不失真。 |
| **全域搜尋 Bar** | 包含「目的地」與「日期」篩選。 | 串接 `trips` API，支援關鍵字與 location 搜尋。 |
| **四步驟引導** | 註冊、選擇加入、建立旅程、快速出團。 | 靜態圖文元件，作為行為引導。 |
| **旅程詳情區** | 呈現標題、標籤 (Tags)、價格與報名按鈕。 | Tags 建議使用圓角邊框小組件。 |
| **關於這趟旅行** | 包含描述與「氛圍 Vibe」區塊。 | Vibe 區塊可使用醒目的背景色與圖示。 |
| **行程規劃 (Timeline)** | 以線條串接各時間點的行程。 | 需對應 `itineraries` 的 `day` 與 `time`。 |
| **團主 Q&A / 注意事項** | 展示團主撰寫的詳細規則。 | 建議支援 Markdown 渲染，讓排版更靈活。 |
| **信任清單 (Trust List)** | 展示真實用戶分享與推薦。 | 動態抓取 `reviews` 與 `site_feedbacks` API。 |

## 💡 UX 建議與優化

1. **情感化設計**：保留樣板中的插畫元素，增加社群溫度。
2. **即時反饋**：點擊「加入旅團」時，應跳出樣板風格的 Modals。
3. **空狀態設計**：搜尋無結果時，使用插畫提示。

## 🚀 技術實作方向

- **前端框架**：使用 **React (Vite)**。
- **樣式架構**：採用 **Bootstrap 5**。
- **後端模擬**：使用 **json-server** 與 **json-server-auth**。
  - `/login`, `/register` 用於身分驗證。
  - `/users`, `/trips`, `/reviews` 等 RESTFUL 排路徑。
  - 所有請求需帶上 JWT Token 進行受保護資源存取。
